# 班级事务管理系统 - 运行指南

## 📋 运行前准备

### 1. 检查环境

首先确认你的电脑上已安装：

**检查Java版本**（需要JDK 1.8或更高）：
```bash
java -version
```

**检查Maven**（可选，如果使用IDE可以不用）：
```bash
mvn -version
```

### 2. 准备MySQL数据库

**确保MySQL服务已启动**：
- Windows: 在服务管理器中查看MySQL服务是否运行
- 或者打开命令行输入：`mysql -u root -p` 能连接成功即可

## 🚀 运行步骤

### 方法一：使用命令行运行（推荐）

#### 步骤1：配置数据库连接

打开文件：`src/main/java/com/classmanagement/config/DatabaseConfig.java`

修改第19-20行的数据库用户名和密码：
```java
config.setUsername("root");        // 改为你的MySQL用户名
config.setPassword("root");         // 改为你的MySQL密码
```

#### 步骤2：创建数据库

打开MySQL命令行或客户端（如Navicat、MySQL Workbench），执行：

```sql
CREATE DATABASE class_management CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

或者直接执行SQL脚本：
```bash
mysql -u root -p < database/init.sql
```

#### 步骤3：编译项目

在项目根目录（`E:\软件工程实训`）打开命令行，执行：

```bash
mvn clean compile
```

#### 步骤4：运行程序

```bash
mvn exec:java -Dexec.mainClass="com.classmanagement.Main"
```

### 方法二：使用IDE运行（更简单）

#### 使用IntelliJ IDEA

1. **打开项目**
   - 打开IDEA，选择 `File` → `Open`
   - 选择项目根目录 `E:\软件工程实训`
   - IDEA会自动识别Maven项目并下载依赖

2. **配置数据库连接**
   - 打开 `src/main/java/com/classmanagement/config/DatabaseConfig.java`
   - 修改第19-20行的用户名和密码

3. **创建数据库**
   - 在IDEA的Database工具中连接MySQL
   - 执行SQL：`CREATE DATABASE class_management CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
   - 或者右键执行 `database/init.sql` 文件

4. **运行程序**
   - 找到 `src/main/java/com/classmanagement/Main.java`
   - 右键点击 → 选择 `Run 'Main.main()'`
   - 或者点击类名旁边的绿色运行按钮 ▶️

#### 使用Eclipse

1. **导入项目**
   - `File` → `Import` → `Existing Maven Projects`
   - 选择项目根目录

2. **配置数据库**（同IDEA步骤2-3）

3. **运行程序**
   - 找到 `Main.java`
   - 右键 → `Run As` → `Java Application`

## ⚠️ 常见问题解决

### 问题1：数据库连接失败

**错误信息**：
```
Communications link failure
或
Access denied for user 'root'@'localhost'
```

**解决方法**：
1. 检查MySQL服务是否启动
2. 检查 `DatabaseConfig.java` 中的用户名和密码是否正确
3. 确认数据库 `class_management` 已创建
4. 如果MySQL端口不是3306，需要修改连接URL中的端口号

### 问题2：找不到主类

**错误信息**：
```
Error: Could not find or load main class com.classmanagement.Main
```

**解决方法**：
1. 先执行 `mvn clean compile` 编译项目
2. 如果使用IDE，刷新Maven项目（右键项目 → Maven → Reload Project）
3. 确保 `target/classes` 目录下有编译好的 `.class` 文件

### 问题3：依赖下载失败

**错误信息**：
```
Could not resolve dependencies
```

**解决方法**：
1. 检查网络连接
2. 配置Maven国内镜像（阿里云）：
   - 编辑 `C:\Users\你的用户名\.m2\settings.xml`
   - 添加镜像配置（如果没有这个文件，创建一个）
3. 清理并重新下载：`mvn clean install -U`

### 问题4：字符编码问题

**解决方法**：
- 确保数据库使用 `utf8mb4` 字符集
- 确保IDE文件编码设置为UTF-8
- 在IDEA中：`File` → `Settings` → `Editor` → `File Encodings` → 全部设置为UTF-8

### 问题5：端口被占用

如果MySQL端口不是默认的3306，修改 `DatabaseConfig.java` 第18行：
```java
config.setJdbcUrl("jdbc:mysql://localhost:你的端口号/class_management?...");
```
